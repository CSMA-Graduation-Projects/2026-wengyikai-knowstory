# 智能用户故事生成系统

基于大语言模型与数据库知识库的需求解析、用户故事生成、任务拆解、代码生成与实验评估一体化平台，支持自然语言需求 / 代码片段双输入，自动完成敏捷需求工程全流程处理。

## 项目简介
本项目是面向软件工程 / 敏捷开发场景的智能需求处理系统，通过大模型结合内置业务知识库，实现从原始需求到用户故事、开发任务、业务代码的全自动生成与质量评估，适用于课程设计、毕设演示、小型项目需求快速拆解等场景。

## 核心功能
- **双输入支持**：自然语言需求输入、代码片段反向解析输入。
- **知识增强**：内置 30+ 通用业务模块知识库（登录 / 注册 / 电商 / 审批 / 权限等），自动匹配增强需求解析。
- **结构化解析**：自动提取需求角色、行为、条件、目标。
- **用户故事生成**：严格遵循「作为 XX，我想要 XX，以便 XX」标准模板。
- **任务拆解**：按前端 / 后端 / 数据库分类，自动生成任务依赖关系。
- **可视化展示**：任务依赖关系图、故事列表、任务清单联动筛选。
- **质量评估**：多维度自动评分 + 优化建议。
- **代码生成**：基于选中故事一键生成 Vue2 + FastAPI + SQLite 代码骨架。
- **结果导出**：支持完整结果导出为 Markdown 文件。

## 技术栈
- **前端**：Vue2 + Element UI + ECharts
- **后端**：FastAPI + SQLAlchemy + SQLite
- **大模型**：DeepSeek 大模型（API 调用）
- **数据库**：SQLite（本地文件库，无需额外部署）

## 快速运行（本地开发）

### 一、环境准备
- **Python 环境**：3.8+
- **Node 环境**：14.x+（前端运行）
- **API 密钥**：DeepSeek API Key（在 `.env` 文件配置）

### 二、后端运行
1.  **克隆 / 下载项目**，进入项目根目录。
2.  **安装 Python 依赖**：
    ```bash
    pip install -r requirements.txt
    ```
3.  **配置环境变量**：
    新建 / 修改 `.env` 文件，填入 DeepSeek API Key：
    ```env
    # DeepSeek API 密钥（必填）
    DEEPSEEK_API_KEY=你的DeepSeek_API_KEY
    # 服务配置
    API_HOST=127.0.0.1
    API_PORT=8000
    DEBUG=true
    ```
4.  **初始化业务知识库**：
    ```bash
    python seed_common_knowledge.py
    ```
5.  **启动后端服务**：
    ```bash
    uvicorn main:app --reload
    ```
    启动成功后：
    - 后端服务地址：`http://127.0.0.1:8000`
    - 接口文档：`http://127.0.0.1:8000/docs`

### 三、前端运行
1.  **进入前端代码目录**（`main.js` 所在目录）。
2.  **安装前端依赖**：
    ```bash
    npm install
    ```
3.  **启动前端项目**：
    ```bash
    npm run serve
    ```
    启动成功后，前端访问地址：`http://localhost:8080`

## 部署说明

### 一、后端生产部署
1.  **关闭调试模式**：修改 `.env` 中 `DEBUG=false`。
2.  **生产启动服务**（推荐使用 gunicorn）：
    ```bash
    # 安装 gunicorn
    pip install gunicorn
    # 启动服务
    gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000
    ```
3.  **守护进程（可选）**：使用 systemd/supervisor 托管服务，保证后台稳定运行。

### 二、前端生产部署
1.  **打包前端代码**：
    ```bash
    npm run build
    ```
2.  **Nginx 部署**：将 `dist` 目录上传至服务器，配置 Nginx 反向代理：
    ```nginx
    server {
        listen 80;
        server_name 你的域名/服务器IP;
        root /path/to/dist;
        index index.html;

        # 后端接口代理
        location /api {
            proxy_pass http://127.0.0.1:8000;
            proxy_set_header Host $host;
        }
    }
    ```

### 三、关键配置说明
- **数据库**：默认使用 SQLite 文件库 `user_story_kb.db`，无需额外部署，自动创建。
- **知识库**：内置模块通过 `seed_common_knowledge.py` 初始化，支持后台接口自定义新增。
- **跨域**：后端已配置全局 CORS，本地开发 / 生产部署均无跨域问题。

## 核心接口文档

1.  **服务健康检查**
    - 请求方式：`GET`
    - 接口地址：`/health`
    - 返回：服务状态、数据库 / 代码生成模块可用性
2.  **知识增强上下文预览**
    - 请求方式：`POST`
    - 接口地址：`/preview_context`
    - 请求参数：`{"requirement": "用户登录系统"}`
    - 返回：匹配的知识库模块、增强上下文文本
3.  **生成用户故事与任务**
    - 请求方式：`POST`
    - 接口地址：`/generate_story`
    - 请求参数：`{"requirement": "自然语言需求"}`
    - 返回：知识上下文、结构化需求、用户故事、任务列表
4.  **生成并评估结果**
    - 请求方式：`POST`
    - 接口地址：`/generate_and_evaluate`
    - 请求参数：`{"requirement": "自然语言需求"}`
    - 返回：生成结果 + 多维度评估分数 + 优化建议
5.  **代码解析生成故事**
    - 请求方式：`POST`
    - 接口地址：`/generate_story_from_code`
    - 请求参数：`{"code": "代码片段"}`
    - 返回：提取的需求、故事、任务、评估结果
6.  **基于故事生成代码**
    - 请求方式：`POST`
    - 接口地址：`/generate_code`
    - 请求参数：`{"requirement": "原始需求", "story": "选中故事", "tasks": [任务列表]}`
    - 返回：前端 / 后端 / 数据库代码骨架
7.  **知识库管理**
    - 查询模块：`GET /knowledge_modules`
    - 新增模块：`POST /knowledge_modules`

## 常见问题 FAQ

- **Q1：启动后端提示 DEEPSEEK_API_KEY 未配置**
  - **A**：未在 `.env` 文件填写有效 DeepSeek API Key，注册 DeepSeek 平台获取密钥后填入即可。
- **Q2：执行 seed_common_knowledge.py 报错**
  - **A**：检查 `database.py`、`knowledge_models.py` 文件是否完整，Python 依赖是否安装齐全。
- **Q3：前端启动后无法访问后端**
  - **A**：确认后端服务已启动在 8000 端口，前端 request 方法中请求地址正确，无端口 / 地址错误。
- **Q4：端口 8000/8080 被占用**
  - **A**：修改后端启动端口 `uvicorn main:app --reload --port 自定义端口`，前端同步修改请求地址。
- **Q5：生成结果为空 / 报错**
  - **A**：检查网络是否可访问 DeepSeek API，需求描述是否清晰，避免过于简短或无意义文本。
- **Q6：打包前端后页面空白**
  - **A**：检查 Nginx 配置是否正确，`dist` 目录路径是否正确，前端路由是否为 hash 模式。
- **Q7：如何新增自定义业务知识库**
  - **A**：通过 `/knowledge_modules` 接口，传入模块名称、别名、要素、任务等参数即可新增。

## 注意事项
- **API 密钥**：必须配置有效的 DeepSeek API Key，否则无法调用大模型生成内容。
- **知识库**：首次运行必须执行 `seed_common_knowledge.py` 初始化，否则知识增强功能失效。
- **端口占用**：确保 8000（后端）、8080（前端）端口未被占用。
- **数据安全**：本地演示使用 SQLite，生产环境可替换为 MySQL/PostgreSQL。
- **模型调用**：代码生成、故事生成均依赖大模型 API，保证网络可访问 DeepSeek 服务。
